# Редакционный процесс журнала

Канонический цикл для каждого дня путешествия.

## 1. Авторские вопросы

Перед финальным отбором фотографий помощник спрашивает автора о дне:

- какие 2-3 эпизода запомнились;
- что удивило;
- какое ощущение осталось;
- какая главная мысль дня.

Ответы сохраняются в:

```text
data/<trip>/<day>-author-notes.json
```

## 2. Анализ фотографий

Фотографии синхронизируются из Cloudinary.

```text
data/<trip>/<day>-photos.json
```

ИИ-анализ фотографий сохраняется в:

```text
data/<trip>/<day>-analysis.json
```

Индивидуальный анализ кэшируется по public_id:

```text
data/<trip>/analysis-cache/<day>/<public_id>.json
```

## 3. Идеи главы

На основе фотографий и авторских заметок формируются варианты идеи главы:

```text
data/<trip>/<day>-ideas.json
```

Выбранная идея фиксируется отдельно:

```text
data/<trip>/<day>-selected-idea.json
```

После выбора идеи все дальнейшие решения должны работать на неё.

## 4. ИИ-предзаполнение редактора

ИИ создаёт предварительный отбор:

```text
data/<trip>/<day>-ai-review.json
```

Это не финальная версия. Это черновик для автора.

Внутри:

- hero;
- story;
- backstage;
- skip;
- подписи;
- редакторские заметки;
- черновая структура главы.

## 5. Авторский редактор

Автор открывает:

```text
editor.html?trip=<trip>&day=<day>
```

Редактор автоматически загружает:

- photos;
- ai-review.

Автор правит статусы и подписи, затем экспортирует:

```text
data/<trip>/<day>-author-review.json
```

Этот файл является авторским решением и имеет приоритет над ИИ.

## 6. Финальный review

После получения author-review помощник собирает:

```text
data/<trip>/<day>-final-review.json
```

Правила:

- author-review — главный источник;
- selected-idea — смысловая основа;
- AI review — только вспомогательный черновик;
- статусы кадров из author-review не менять без крайней необходимости;
- текст улучшать, факты проверять, географию не придумывать.

## 7. Preview

Preview открывается так:

```text
preview.html?trip=<trip>&day=<day>
```

Он читает файлы по приоритету:

1. final-review;
2. author-review;
3. ai-review;
4. старый review как fallback.

## 8. Публикация

Публикация на основной сайт происходит только после явного утверждения автора.

Черновики, редактор и preview не должны быть доступны из основной навигации сайта.
