# Редакционный процесс журнала

Канонический цикл для каждого дня путешествия.

1. Авторские вопросы: помощник спрашивает автора о дне и сохраняет ответы в data/<trip>/<day>-author-notes.json.

2. Анализ фотографий: Cloudinary sync создаёт data/<trip>/<day>-photos.json, затем ИИ-анализ сохраняется в data/<trip>/<day>-analysis.json.

3. Идеи главы: варианты идей сохраняются в data/<trip>/<day>-ideas.json, выбранная идея фиксируется в data/<trip>/<day>-selected-idea.json.

4. ИИ-предзаполнение редактора: создаётся data/<trip>/<day>-ai-review.json. Это не финальная версия, а черновик для автора.

5. Авторский редактор: автор открывает editor.html?trip=<trip>&day=<day>, правит статусы и подписи, затем экспортирует data/<trip>/<day>-author-review.json. Этот файл имеет приоритет над ИИ.

6. Финальный review: после получения author-review помощник собирает data/<trip>/<day>-final-review.json. Статусы из author-review не менять без крайней необходимости. Текст можно улучшать, факты нужно проверять.

7. Storyboard: после final-review создаётся data/<trip>/<day>-storyboard.json. Это сценарий главы: сцены, эмоциональная дуга, ритм текста и роль блоков фотографий. Storyboard нужен, чтобы глава была историей, а не галереей с подписями.

8. Preview: preview.html?trip=<trip>&day=<day> читает final-review, затем author-review, затем ai-review, затем старый review как fallback.

9. Публикация: только после явного утверждения автора. Черновики, редактор и preview не должны быть доступны из основной навигации сайта.
