# Автоматические уведомления о публикациях

После утверждённой публикации обновляется файл:

`data/notifications/latest.json`

Любое изменение этого файла в ветке `main` запускает workflow:

`.github/workflows/send-publish-notification.yml`

Он отправляет push всем пользователям сегмента `Subscribed Users` в OneSignal.

## Формат

```json
{
  "trip": "kyrgyzstan-2026",
  "day": "day08",
  "title": "Новая глава журнала",
  "message": "«Пора домой» — последний день путешествия по Кыргызстану.",
  "url": "https://owntravel.ru/day08.html",
  "icon": "https://owntravel.ru/icons/icon-192.png",
  "published_at": "2026-07-11T12:00:00.000Z"
}
```

## Секрет

В настройках репозитория должен существовать Actions secret:

`ONESIGNAL_REST_API_KEY`

Секрет никогда не хранится в файлах репозитория.

## Правило публикации

Обновлять `latest.json` только после того, как:

1. публичная глава создана;
2. навигация и страница путешествия обновлены;
3. опубликованный URL проверен;
4. пользователь дал прямую команду на публикацию.

Повторное изменение `latest.json` повторно отправит уведомление, поэтому не использовать его для исправления текста уже отправленного push.
